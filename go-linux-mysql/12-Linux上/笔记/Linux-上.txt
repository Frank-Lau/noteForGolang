shell / bash：

	命令解析器：自动解析用户键入终端的命令。调用同名可执行文件。将结果展现在终端中。

	自动运行于终端中。

绝对路径：从 linux 的根目录 找寻文件的访问路径。

相对路径：相对于当前“shell”工作目录位置。

	“.” 当前目录位置

	“..”当前目录的上一级目录位置。

Linux操作系统中，看到所有的东西，都对应文件。	―― 所见皆文件。

Linux系统主要目录:

	"/": 根目录，存放系统目录。

	"/bin":	二进制的可执行文件。对应各个系统命令。

	"/dev": 	设备所对应的文件。

	"/etc":	系统配置文件。或第三方安装软件配置。

	"/home":	普通用户的 “家目录”（宿主目录）		“家” !=“home”

		root 用户 的家目录 --- /root

	"/lib":	系统使用的函数库存放的位置。

	"/tmp":	存放用户程序运行期间产生的 临时文件。

	"/var":	存放系统运行过程中，经常发生变化的程序目录。	/var/lib/mysql 	MySQL数据库文件。

ls -l:	显示 指定目录下，所有文件的 详细信息。

	文件属性（10个字符）  硬链接计数   所有者	  所属组    大小    创建、最后修改时间	文件名 （7部分）

	1. 文件属性：
		
		文件类型：  第1个字符。

		Linux系统中不以文件后缀名作为区分文件类型依据！！！

			1. 普通文件。	“-”		.mp3/avi/rmvb....

			2. 目录文件。 	“d”

			3. 软连接文件。	“l”

			4. 字符设备文件。“c”

			5. 块设备文件。	“b”

			6. 管道文件。	“p”

			7. 套接字文件。	“s”

			8. 未知 unknown

			1-3 实际占用磁盘存储得文件类型。  4-7 不占用磁盘存储，属于伪文件。

		文件访问属性：后 9 个字符。分 3 组。

			文件所有者 读、写、执行权限。  rwx   没有权限用“-”

			文件所属组 读、写、执行权限。

			其他人 读、写、执行权限。
	2. 硬链接：
		ln   源文件名   硬链接文件名。	―― ln a.c a.hard

		原文件和其对应的硬链接文件，同步更新。

		硬链接计数：统计该文件的访问途径。

		删除文件时，将文件所对应的硬链接计数 -1 。 当文件所对应的硬链接 减为 0 时，该文件具备被清除的条件。

		ls -i ： 查看指定文件的 inode（i节点）： 磁盘上一个文件的唯一存储标识。

			也可以使用 stat 获取 i节点。eg stat a.c

	3. 所有者、所属组：当一个用户被创建，自动创建该用户对应的用户组，并且该用户属于用户组。

	4. 大小： 对于目录而言，是该目录对应的 存储空间的大小,而不是实际大小,文件为实际大小。

	5. 时间：创建、最后修改时间

ls -ld:	查看目录本身的详细信息。  不添加 d， 查看目录内容的详细。

通配符：
	*： 0-N个任意字符。

	？：表示一个字符。

重定向：
	> : 将输出重定向到 指定文件。文件不存在自动创建，文件存在覆盖源文件内容。

	>>: 追加重定向,不覆盖,在文件内容末尾追加。

	more： 按百分比查看一个文件。  回车一行，空格一屏，q 退出。

	less：分屏查看大文件，不显示百分比, 回车一行，空格一屏，q 退出。 支持 ↑↓

	head： 语法： head -N 文件名： 	从文件起始显示 N 行。

	tail： 语法： tail -N 文件名： 	从文件结尾处向前显示 N 行

切换用户：
	切换root： sudo su	不改变当前shell的工作目录位置。  等价于： su root 。

		 su - root   改变当前shell的工作目录位置 到 root 的家目录。

	切换普通用户：

		su - 用户名：  改变当前shell的工作目录位置 到 新用户的家目录。切换用户。

		su 用户名：不改变当前shell的工作目录位置。切换用户。

	sudo：关键字。让普通用户，临时获得 root用户权限。 执行完当前命令即失效。

创建用户：

	sudo  adduser  新用户名 

	指定 密码。 一次 回车。

	在 /home/ 创建 新用户宿主目录。  在 /etc/passwd 文件中，添加 新用户的一条描述信息。

删除用户：

	sudo deluser 用户名  	在 /etc/passwd 文件中，抹除用户记录。不会删除 用户的家目录。	

创建用户组：

	sudo addgroup 新组名

	在 /etc/group 文件中，添加 新用户组的一条描述信息。

删除用户组：
	sudo delgroup 组名

chown：修改文件所有者：
	
	sudo chown 新用户名   文件名

chgrp ：修改文件所属组：
	
	sudo chgrp 新组名	文件名

一次性设置所有者、所属组： 	

	sudo chown 新用户名:新组名	文件名。
